## 2. 推荐实机环境

| 项目 | 要求 |
|---|---|
| 操作系统 | Ubuntu 22.04 amd64 |
| ROS 版本 | ROS2 Humble |
| 仿真器 | Gazebo Classic |
| 虚拟机 | VirtualBox 可用，建议开启 3D Acceleration |
| CPU | 4 核以上，推荐 8 核 |
| 内存 | 8 GB 最低，推荐 10-16 GB |
| 显存 | 128 MB 以上 |

默认运行策略面向 VirtualBox 稳定性：

```text
gui:=true
rviz:=true
performance:=low
route:=safe
dynamic_obstacles:=false  # 建图阶段
```

## 4. 推荐运行顺序

### 4.1 首次仿真检查

```bash
ros2 launch project bringup.launch.py mode:=sim gui:=true rviz:=true performance:=low dynamic_obstacles:=true
```

### 4.2 自动建图短路线验证

```bash
ros2 launch project bringup.launch.py mode:=slam_auto route:=safe gui:=true rviz:=true performance:=low dynamic_obstacles:=false
```

`route:=safe` 只用于首次验证。通过后正式建图应切换到 `coverage`。

### 4.3 自动建图正式路线

```bash
ros2 launch project bringup.launch.py mode:=slam_auto route:=coverage gui:=true rviz:=true performance:=low dynamic_obstacles:=false
```

如果地图边角或货架通道还不够完整，运行更长路线：

```bash
ros2 launch project bringup.launch.py mode:=slam_auto route:=extended gui:=true rviz:=true performance:=low dynamic_obstacles:=false
```

自动巡航结束后不会自动保存地图。请等待 10-30 秒，检查 RViz 地图质量后手动保存。

### 4.4 保存地图

```bash
mkdir -p ~/warehouse_ws/src/project/maps
ros2 run nav2_map_server map_saver_cli -f ~/warehouse_ws/src/project/maps/warehouse_map
```

### 4.5 启动导航与动态避障

```bash
ros2 launch project bringup.launch.py mode:=nav gui:=true rviz:=true performance:=low dynamic_obstacles:=true map:=~/warehouse_ws/src/project/maps/warehouse_map.yaml
```

### 4.6 多目标点导航

```bash
ros2 run project waypoint_nav.py
```

## 5. 性能模式

```text
normal:
  LiDAR 1080 samples @ 10 Hz
  RGB-D 640x480 @ 15 Hz
  IMU 100 Hz

low:
  LiDAR 540 samples @ 8 Hz
  RGB-D 320x240 @ 8 Hz
  IMU 50 Hz
```

VirtualBox 下默认使用 `performance:=low`。如果运行流畅，再尝试 `performance:=normal`。

## 6. 一句话流程

```text
mode:=sim 检查仿真和 GUI
-> mode:=slam_auto route:=safe 验证自动巡航
-> mode:=slam_auto route:=coverage 正式自动建图
-> map_saver_cli 手动保存地图
-> mode:=nav dynamic_obstacles:=true 启动导航与动态避障
-> waypoint_nav.py 执行多目标点导航
```

## 7. 注意事项

- 建图阶段建议 `dynamic_obstacles:=false`，避免动态叉车污染地图。
- 导航演示阶段建议 `dynamic_obstacles:=true`，展示动态避障。
- `route:=safe` 不是正式建图路线，只用于首次验证。
- `route:=coverage` 是正式建图默认路线。
- `route:=extended` 用于追求更完整、更精细的地图。
- `gui:=true` 会正常打开 Gazebo GUI；`gui:=false` 才使用 headless/xvfb。
- 3D RTAB-Map 仅作为扩展，不进入 VirtualBox 主演示路径。
