## 2. 实机运行环境要求

真正运行 ROS/Gazebo 项目需要以下环境：

| 项目 | 要求 |
|---|---|
| 操作系统 | Ubuntu 22.04 |
| ROS 版本 | ROS2 Humble |
| 仿真器 | Gazebo Classic |
| Python | Python 3.10+ |
| 内存 | 2D 建图/导航建议 8 GB 以上；RTAB-Map 3D 建图建议 16 GB 以上 |

## 3. 推荐工作空间结构

建议把项目放入 ROS2 工作空间：

```text
warehouse_ws/
└── src/
    └── project/
```

如果项目当前在 macOS 上，需要复制 `project/` 文件夹到 Ubuntu 的：

```bash
~/warehouse_ws/src/project
```

## 4. 安装依赖

在 Ubuntu 22.04 中执行：

```bash
sudo apt update
sudo apt install -y \
  ros-humble-gazebo-ros-pkgs \
  ros-humble-xacro \
  ros-humble-robot-state-publisher \
  ros-humble-joint-state-publisher \
  ros-humble-teleop-twist-keyboard \
  ros-humble-cartographer \
  ros-humble-cartographer-ros \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-rtabmap-ros \
  ros-humble-rtabmap
```

然后在工作空间根目录执行：

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
rosdep install --from-paths src --ignore-src -r -y
```

## 5. 编译项目

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
colcon build --symlink-install
source install/setup.bash
```

如果使用 conda 并遇到 `GLIBCXX` 相关错误，先退出 conda：

```bash
conda deactivate
source /opt/ros/humble/setup.bash
source ~/warehouse_ws/install/setup.bash
```

## 6. 推荐运行顺序

### 6.1 启动 Gazebo 仿真和机器人

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch project spawn.launch.py
```

### 6.2 新终端启动键盘遥控

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

### 6.3 启动 2D SLAM 建图

新终端执行：

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch project slam.launch.py
```

然后使用键盘遥控机器人遍历仓储环境。

### 6.4 保存地图

建图完成后执行：

```bash
mkdir -p ~/warehouse_ws/src/project/maps
ros2 run nav2_map_server map_saver_cli -f ~/warehouse_ws/src/project/maps/warehouse_map
```

保存成功后应生成：

```text
warehouse_map.yaml
warehouse_map.pgm
```

### 6.5 启动 AMCL + Nav2 导航

注意：必须先完成地图保存，否则导航会因为找不到地图失败。

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch project nav.launch.py map:=~/warehouse_ws/src/project/maps/warehouse_map.yaml
```

### 6.6 运行多目标点导航

新终端执行：

```bash
cd ~/warehouse_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 run project waypoint_nav.py
```

## 7. 3D 建图

RTAB-Map 需要更高内存，建议 16 GB 以上：

```bash
cd ~/warehouse_ws

source install/setup.bash
ros2 launch project rtabmap.launch.py
```

## 8. 一句话流程

```text
spawn.launch.py 启动仿真
-> teleop_twist_keyboard 遥控机器人
-> slam.launch.py 建图
-> map_saver_cli 保存地图
-> nav.launch.py 加载地图并导航
-> waypoint_nav.py 执行多目标点导航
```

## 9. 注意事项

- 不要一开始直接运行 `nav.launch.py`，因为项目默认需要先生成 `maps/warehouse_map.yaml`。
- `spawn.launch.py` 是启动仿真和机器人，不负责建图或导航。
- `slam.launch.py` 负责 Cartographer 2D 建图。
- `nav.launch.py` 负责地图加载、AMCL 定位和 Nav2 导航。
- `waypoint_nav.py` 需要在 Nav2 已经启动并可用后运行。
- 当前 macOS 只能用于静态测试，不能替代 Ubuntu ROS2 实机验证。
